# Capture [Захват]

**Capture** — это стратегическая настольная игра для 2–4 игроков (против людей или ИИ), в которой участники расширяют свои владения, захватывают территории противников и нейтральные зоны с помощью прямоугольников, размер которых определяется броском двух кубиков.

Игра реализована на чистом JavaScript + HTML5 Canvas, поддерживает **режим ботов**, **полную симуляцию**, **адаптивный размер поля**, **настройки для дальтоников** и работает в любом современном браузере.

---

## 🎲 Правила игры

### Цель
Стать последним оставшимся на поле игроком или набрать максимальную площадь территории к моменту полного заполнения поля.

### Игровое поле
- Размер поля: по умолчанию **39×28 клеток** (размер клетки — 20 px). Можно выбрать **кастомный размер** от 4×4 до 100×100 (минимальный размер зависит от стартового квадрата).
- В начале игры каждый игрок получает **стартовый квадрат** (размер от 1×1 до 6×6, настраивается) в своём углу:
  - 🔴 Красный — левый верхний угол.
  - 🔵 Синий — правый верхний угол.
  - 🟢 Зелёный — левый нижний угол.
  - 🟡 Жёлтый — правый нижний угол.
- Для 2 игроков стартовые позиции: красный (левый верх) и синий (правый низ).
- Все клетки поля изначально нейтральны (серые), кроме стартовых.

### Ход игрока
1. **Бросок кубиков**  
   Игрок нажимает кнопку «Бросить кубики». Выпадают два числа (от 1 до 6). Эти числа определяют **стороны прямоугольника**, который можно поставить на поле (порядок не важен: 3×4 = 4×3).

2. **Размещение прямоугольника**  
   - Можно **занять пустую область** (расширение), если она **прилегает к вашей территории по общей стороне** (не углом).  
   - Можно **захватить чужой прямоугольник** (вражеский или нейтральный), если он **прилегает к вашей территории** и полностью совпадает с выпавшим размером.  
   - Захваченный прямоугольник меняет владельца.

3. **Завершение хода**  
   После размещения (или пропуска) ход переходит к следующему игроку.  
   - Если после захвата или расширения была **накрыта стартовая клетка другого игрока**, этот игрок **выбывает**, а вся его территория **перераспределяется** между оставшимися (приоритет у захватившего стартовую клетку).

### Условия окончания игры
- **Победитель по выбыванию** — последний игрок, оставшийся на поле.
- **Победитель по заполнению** — если поле полностью покрыто прямоугольниками (нейтральные клетки отсутствуют), игра заканчивается, и побеждает игрок с наибольшей суммарной площадью своих прямоугольников.

### Пропуск хода
Игрок может пропустить ход, не бросая кубики (кнопка «Пропустить ход»). Ход переходит следующему.

## 🤖 Боты и симуляция

- **Режим ботов** (включён по умолчанию): игроки 2, 3 и 4 управляются ИИ. Вы играете за игрока 1.
- **Режим симуляции**: все игроки становятся ботами. Идеально для наблюдения за партией или тестирования логики.
- ИИ выбирает наилучший доступный прямоугольник (максимальная площадь, приоритет захвата стартовой клетки противника).

## 🖥️ Управление

### Мышь / Тачпад
- **Клик по полю** — установить угол прямоугольника (первый клик) и противоположный угол (второй клик).
- **Кнопки**:
  - 🎲 «Бросить кубики»
  - ⏭ «Пропустить ход»
  - 🔄 «Новая игра» (также на левой панели)
  - ⚙️ «Шестерёнка» — открыть настройки игры

### Клавиатура
| Клавиша           | Действие                              |
|-------------------|---------------------------------------|
| `Enter`           | Бросить кубики (если разрешено)       |
| `Esc`             | Отменить выбор прямоугольника         |
| `S`               | Пропустить ход                        |

## ⚙️ Настройки (меню ⚙️)

В модальном окне доступны следующие параметры:

| Параметр                | Описание                                                                 |
|-------------------------|--------------------------------------------------------------------------|
| **Количество игроков**  | 2 или 4. Влияет на расположение стартовых квадратов и активных игроков.  |
| **Размер стартовых квадратов** | От 1×1 до 6×6. Также влияет на **минимальный размер поля** (2×старт+6). |
| **Размер поля**         | Стандартный (39×28) или кастомный (ширина и высота от 4 до 100).         |
| **Режим ботов**         | Вкл/выкл — игроки 2,3,4 (или игрок 2 при 2 игроках) становятся ИИ.      |
| **Режим симуляции**     | Все игроки — боты. Отключает управление и позволяет наблюдать.           |
| **Цветовой режим**      | Обычный / Протанопия / Дейтеранопия / Тританопия (для дальтоников).      |

> При изменении любого параметра игра **перезапускается** с новыми настройками.

## 📊 Интерфейс

- **Левая панель** — счёт игроков (количество занятых клеток), счёт нейтральных клеток, номер хода, таймер партии.
- **Центр** — игровое поле с сеткой. На каждом прямоугольнике отображается его площадь (количество клеток).
- **Нижняя панель** — индикатор текущего игрока, выпавшие кубики, кнопки управления и сообщения о событиях.
- **Поле под канвасом** — отображает текущий размер поля и активный режим (боты / симуляция).

## ⚙️ Технические детали

- **Canvas** — динамический размер (ширина = COLS × 20 px, высота = ROWS × 20 px).
- **Логика** — чистая реализация на JavaScript без внешних библиотек.
- **Адаптивность** — интерфейс подстраивается под мобильные устройства (медиа-запросы).
- **Сохранение состояния** — не предусмотрено (игра не использует localStorage).
- **Таймер** — отсчитывает общее время партии, останавливается при завершении игры.

## 🧩 Особенности реализации

- **Проверка прилегания** — прямоугольник должен касаться территории игрока **по стороне** (не только углом).
- **Перераспределение территории выбывшего**:
  1. Все прямоугольники выбывшего проверяются на прилегание к территориям живых игроков.
  2. Приоритет получает игрок, который накрыл стартовую клетку («убийца»).
  3. Остальные прямоугольники распределяются между другими живыми игроками по **близости к стартовой клетке выбывшего** (евклидово расстояние).
  4. Если прилегающих нет — прямоугольник становится нейтральным.
- **Захват нейтральной территории** — требует прилегания к своей территории.
- **ИИ ботов** — перебирает все возможные прямоугольники (новые и существующие) и выбирает ход с максимальным «весом»: площадь + бонус за накрытие стартовой клетки противника.
- **Визуальная подсказка** — при выборе прямоугольника подсвечивается зелёным (размер совпадает с кубиками) или красным (не подходит).
- **Цветовые режимы** — корректируют палитру для лучшей различимости (жёлтый игрок меняется на фиолетовый при протанопии/дейтеранопии).

---

## 📝 Примечания

- Игра поддерживает 2 или 4 игроков. При выбывании игроков оставшиеся продолжают.
- Нейтральные прямоугольники можно захватывать только целиком (не частями).
- Захваченный вражеский прямоугольник сразу меняет владельца, площадь суммируется.
- В режиме симуляции **все** кнопки управления блокируются — вы только наблюдатель.
